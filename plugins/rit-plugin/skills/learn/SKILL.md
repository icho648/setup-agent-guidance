# Learn

用最轻量但足以保证学习真实发生的方式，把现实目标转化为经过证明、可以迁移
的能力。讲过、看过、生成过材料和自我感觉良好都不等于掌握。

默认使用用户当前使用的语言教学。

## 核心契约

- 每次只围绕一个清晰、可观察的学习成果推进。
- 不把课程交付、项目完成、文件存在或智能体代做当作学习者能力证据。
- 先让学习者作出关键判断或尝试，再按需要揭示图、代码和完整示范。
- 学习进度与能力等级分开维护；只有证据改变时才更新能力等级。
- 对话是主要教学界面，真实环境是首选练习场，代码编辑器用于阅读完整源码。

## 持久学习状态

所有持久化教学状态都放在当前工作区的 `.learning/` 下，不要写入业务工作区
根目录：

- `MISSION.md`：现实目标、成功标准、约束和非目标。
- `MASTERY.md`：能力地图、学习进度、目标等级、证据和下一个挑战。
- `RESOURCES.md`：少量、经过注释的可信资料与真实反馈渠道。
- `NOTES.md`：教学偏好、无障碍需求和不属于掌握证据的工作笔记。
- `learning-records/*.md`：已经证明且会改变未来教学假设的能力或误解修正。
- `reference/*`：值得反复查阅的压缩知识，如心智模型、术语表和检查清单。
- `lessons/*`、`assets/*`：仅在可复用课程或交互明显改善学习时创建。

按需创建文件，不要一次搭完整套结构，不复制已有规范、源码或路线。具体格式
见[学习工作区格式](references/workspace-formats.md)。

必须分开记录：

- **学习进度**：`待开始`、`进行中`、`已完成`，表示是否达到约定的学习成果。
- **能力等级**：`未验证`、`识别`、`引导完成`、`独立完成`、`迁移运用`，
  表示当前证据支持学习者做到什么程度。

“讲过”默认只把学习进度推进到“进行中”。只有用户明确确认，或完成了事先
约定的验收产物，才能记为“已完成”；这仍不自动提高能力等级。

## 开始或继续学习

1. 检查 `.learning/`、相关工作区产物、已有计划和当前请求。能从环境发现的
   事实先自行查找，不要询问用户。
2. 明确学习使命。目标具体时直接形成简短草案；缺失信息会实质改变路线时才
   提问。大幅修改已有使命前必须确认。
3. 新计划先用一个简短、真实的解释、判断、追踪、演算、操作或诊断建立基线，
   不先发知识问卷或完整课程。
4. 建立或更新 `MASTERY.md`，优先描述最近一到三个可观察能力；更远路线保持
   粗粒度。
5. 为本轮选定一个成果、通过证据和最轻教学方式。

如果学习者要求回到更早章节或改变当前焦点，尊重该选择并更新当前挑战；不要
清空历史状态，也不要把重新听讲当成重新验证。

### 导入已有计划

当 `.learning/` 尚不存在，而工作区或用户材料中已有路线与进度时：

- 以来源和日期注明暂定基线，不要求用户为了初始化而重学。
- 已接触但没有验收证据的主题记为“进行中”；只有用户确认或证据满足验收时
  才记为“已完成”。
- 能力等级只按学习者证据填写，不用 AI 总结、课程标记或项目状态覆盖它。
- 证据注明 `当前验证`、`历史观察`、`用户陈述` 或 `材料存在`。
- 只询问会改变当前路线的未知项，并以一次高价值回忆或真实任务继续学习。

## 单次学习闭环

根据风险和难度调整强度，但保持以下顺序：

1. **先回忆**：再次讲解前，先检索一到三个相关知识点；第一次接触时跳过。
2. **定义成果**：说明本轮结束时能做什么，以及什么表现算通过。
3. **最少但充分讲解**：只给足以开始尝试的地图、概念、例子、资料或代码入口；
   不把“最少”误解为对关键概念保持浅层或统一颗粒度。
4. **真实尝试**：让学习者预测、判断、操作、诊断、设计或解释关键步骤。
5. **精确反馈**：指出正确部分、证据缺口和最小修正动作。
6. **迁移验证**：改变一个关键条件，要求重新判断、适配、排错或解释取舍。
7. **更新状态**：分别记录学习进度、能力等级、证据口径和下一个挑战。

学习者卡住时逐级提供帮助：重述目标或现象 → 指向概念、位置或步骤 → 给出
局部例子或代码片段 → 用户明确要求或继续挣扎已无学习收益时再完整示范。

## 教学模式

按成果选择并自然组合以下模式，不要求学习者管理这些名称：

- **建立地图**：最短领域地图、核心词汇和学习顺序。
- **理解概念**：只解释完成下一挑战所需的知识。
- **检索测验**：用回忆、预测、选择、排序或情境判断暴露缺口并立即反馈。
- **真实练习**：在真实环境完成任务，建立紧密反馈循环。
- **故障诊断**：观察失败，让学习者依靠状态、日志、测试或测量定位原因。
- **迁移表达**：复述、适配新情境、解释取舍、教学或接受追问。

## 重点深入

先用合适的统一颗粒度建立全局地图，再按学习者意图、当前证据和概念重要性
选择一到两个认知瓶颈重点深入。重点深入是单独的教学环节，不把整个章节机械地
维持在同一深度，也不等于无边界地增加细节。

以下条件并列；满足任意一项就进入重点深入：

- 用户明确要求深入某个主题、查看实现、展开细节或进行更完整的解释。
- 用户对当前讲解提出疑惑、追问、反对或纠正，说明现有颗粒度不足。
- 用户表示没听懂、想知道为什么，或要求换一种方式解释。
- 学习者的回答、操作或判断暴露错误、犹豫、边界混淆或不稳定理解。
- 概念直接决定当前学习成果、使命成功标准或当前挑战能否完成。
- 概念涉及跨进程、异步、持久化、安全边界或隐蔽且容易误诊的失败模式。
- 概念是多个后续能力的共同前置知识，当前缺口会持续放大。

用户表达的要求和疑惑与模型根据证据作出的判断具有同等触发权。用户主动提问
时先回答并解决疑惑，不把完成新的挑战作为获得解释的门槛；不要因为模型认为
“已经讲过”或“不是主线”而跳过用户要求的深入。

进入重点深入后：

1. 明确本次要解决的一个具体困惑或可观察成果，以及为什么此处需要放大。
2. 说明状态拥有者、运行时顺序、关键边界、取舍和失败表现；只补充理解该瓶颈
   所需的相邻知识。
3. 选择真正有助于理解的图、表、对比、例子或模拟。代码库学习还要先验证当前
   源码，再按“图或表 → 文件路径 → 关键代码 → 学习者任务”组织。
4. 使用引导追踪、预测、解释或小型诊断确认理解，并用变化场景检查能否迁移。
5. 学习者能够稳定区分相邻概念、解释关键因果或完成当前任务后返回章节主线；
   用户明确表示已经足够或希望停止时也结束深入。

重点深入本身不构成掌握证据。用户只要求解释而没有完成关键判断时，记录教学
偏好或工作笔记即可，不提高能力等级。

## 教学呈现与代码导航

排版只服务于理解。先选择最小、最有信息量的表达：

| 需要讲清的关系 | 优先形式 |
|---|---|
| 单一事实、一步动作或两三个节点的短链路 | 短段落或紧凑箭头 |
| 三项以上依赖、跨层调用、分支或数据流 | Mermaid `flowchart` |
| 请求、事件、并发或异步先后关系 | Mermaid `sequenceDiagram` |
| 生命周期、状态转换和失败恢复 | Mermaid `stateDiagram-v2` |
| 重复字段、职责映射、方案或状态对比 | 表格 |
| 源码本身是事实边界 | 可点击路径与少量关键代码 |

使用图形和代码时遵守以下协议：

1. 图、表和代码都服务于本轮同一个学习成果，不做装饰性可视化。
2. Mermaid 使用真实边界和简短标签；带标点或括号的节点文字使用引号。图后用
   一两句话点明最重要的边界、状态拥有者或失败模式。
3. 代码链路按真实运行顺序给出少量文件。先验证当前源码，再使用带行号的可点击
   绝对路径；不要让学习者在目录树中猜入口。
4. 路径旁只贴理解边界所需的代码，如函数签名、核心数据结构、状态转换、
   `await` 调用、任务派发或错误处理。省略无关实现，并明确标记省略。
5. 推荐组织为：**图或表 → 文件路径 → 关键代码 → 学习者任务**。简单主题可
   跳过图表，但不要只贴路径或大段源码。
6. 首次尝试前只提供不会直接泄露答案的入口和必要上下文；尝试后再逐步揭示
   精确代码与完整调用链。

## 证据与能力判级

重要能力尽量使用多个相互独立的信号：

- **结果证据**：可观察成果或表现。
- **过程证据**：能解释结果如何产生。
- **验证证据**：测试、日志、测量、评审或可复现对比支持结果。
- **迁移证据**：能处理变化场景、排错、教学或为取舍辩护。

测验用于检索和诊断，不自动等于掌握。单次选择题通常最多支持“识别”；解释
判断依据、排除错误选项或在提示下完成任务可支持更高过程证据；能独立完成并
验证真实任务才支持“独立完成”；适配新情境、排错或教学才支持“迁移运用”。

| 能力等级 | 未来会话可以安全假设什么 |
|---|---|
| 未验证 | 尚无足以判断个人能力的证据 |
| 识别 | 在提示下能辨认并解释 |
| 引导完成 | 借助提示、检查表或范例能完成 |
| 独立完成 | 不依赖任务专用提示即可完成并验证 |
| 迁移运用 | 能适配、排错、教学或为新情境中的选择辩护 |

目标等级由学习使命决定，并非每项能力都必须达到“迁移运用”。软件学习中的
证据通常包括可观察行为、调用链解释、日志或测试验证，以及代码差异和回归结果。

证据来源必须标明：

- **当前验证**：本轮直接观察到学习者的解释、操作、结果或反馈。
- **历史观察**：可信历史记录中观察到过，本轮未重新验证。
- **用户陈述**：学习者报告已经做过或掌握。
- **材料存在**：课程、代码、笔记、测试或产物存在。

提高能力等级时优先使用当前验证和可复现的历史观察。智能体生成或运行成功的
产物默认只算“材料存在”，除非学习者本人同时完成了关键判断、操作或解释。
只有证据建立了新的能力下限、纠正了重要误解或改变使命时才创建学习记录。

## 事实、资料与间隔练习

专业、可能变化或会影响结论的事实按以下优先级取证：真实环境和可观察行为 →
一手资料、官方文档、源码和规范 → 必要时使用可信二手资料。明确区分事实、
推断和建议；工作区源码已经是一手资料时，不强制外部引用。

精简维护 `RESOURCES.md`，不要把它变成链接仓库。后续会话先检索一到三个相关
知识点；经过间隔后用变化问题复查薄弱或重要能力，再逐步交错相邻技能。

## 产物与交互边界

按以下优先级选择载体：真实环境中的操作或诊断 → 对话练习与反馈 → Markdown
参考资料或检查表 → 交互式 HTML → 学习应用。只有后一种载体能显著缩短反馈
循环或支持重复练习、分支、模拟、可视化、打印时才升级，不为排版或源码导航
创建 HTML。

- 对话优先，只落盘持久状态或真正值得复用的材料。
- 内容主要用于查阅时，创建紧凑的 `reference/*`，不要再复制学习路线或状态。
- 需要可复用课程时默认使用 Markdown；需要即时自动反馈、重复练习、分支或
  模拟时才创建 HTML。创建前完整阅读
  [交互式课程规范](references/interactive-lessons.md)，并复用
  `assets/interactive-lesson/`。
- 浏览器中的分数、完成标记、自评和 `localStorage` 只用于即时反馈；除非用户
  明确提交结果或智能体直接观察到表现，否则不得更新 `MASTERY.md`。

### 交互式学习应用

只有实时材料导航、文件变化反馈、模拟或跨会话练习明显改善长期学习时，才在
`.learning/` 中创建应用。用户未明确要求时，先说明收益与成本并询问是否创建。

- 默认避免依赖、`node_modules`、构建和打包；优先浏览器能力与运行环境内置模块。
- 学习代码库时实时、只读地读取当前源码和学习状态，不把源码静态复制进应用。
- 访问范围采用最小白名单，拒绝密钥、环境文件、目录穿越和无关工作区内容。
- 除非用户明确要求真实练习，不修改业务源码、数据库或运行中服务。
- 外部库可使用固定版本的远程脚本、ES Module 或本地文件；根据离线、可复现、
  网络和安全要求选择，不把某一种方案设为通用强制结构。
- 提供最短启动方式，并验证核心交互、实时读取、变化响应和安全边界；不要自动
  提交或部署。

## 学习者控制与高风险边界

本技能用于学习，不悄悄接管任务。学习与实际交付并存时，明确哪些步骤由智能体
完成、哪些关键判断应由学习者尝试。根据不确定性、后果和可逆性提高教学强度，
而不是只看主题大小；可选的重型诊断、项目实践或面试循环先说明收益与成本。

医疗、法律、财务、身体操作等高风险领域采用相应安全措施和合格现实指导，不
把工作区课程或能力等级当作专业认证。

## 结束一次会话

保持简短，只说明：

- 学习者实际证明了什么；
- 哪些主题的学习进度已完成，但能力等级仍未达到目标或尚未验证；
- 使用了哪些证据；
- 推荐的一个下一步挑战。

文件保存长期状态，对话中不要再写一份会话流水账。
