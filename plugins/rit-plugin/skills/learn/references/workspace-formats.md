# 学习工作区格式

只创建当前学习计划真正需要的文件，并保持简洁。

## `.learning/MISSION.md`

```md
# 学习使命：<主题或能力>

## 为什么
<这次学习服务于哪个具体的现实结果。>

## 成功标准
- <可观察的能力及其使用情境>
- <可观察的能力及其使用情境>

## 约束
- <时间、工具、安全、无障碍或学习偏好>

## 非目标
- <本阶段明确推迟的相邻内容>
```

一个工作区只保留一个当前学习使命。实质修改已有使命前先确认。

## `.learning/MASTERY.md`

```md
# 能力地图

| 能力 | 学习进度 | 目标等级 | 能力等级 | 证据 | 下一个挑战 |
|---|---|---|---|---|---|
| <可观察能力> | 进行中 | 独立完成 | 引导完成 | 历史观察：<路径、结果或日期> | <一个任务> |
```

允许的学习进度为：`待开始`、`进行中`、`已完成`。
允许的能力等级为：`未验证`、`识别`、`引导完成`、`独立完成`、
`迁移运用`。

能力描述必须可以观察。优先写“根据日志和状态诊断一次失败的数据库迁移”，
不要写“理解数据库”。

学习进度回答学习任务是否达到约定验收：未进入主题为“待开始”，已经讲过、
接触过或正在练习但尚未验收为“进行中”，用户明确确认或验收通过才是
“已完成”。“进行中”不意味着必须从头重讲。

能力等级回答学习者目前能做什么，只能由能力证据提升。“已完成”不自动
等于达到目标等级；后续通过回忆、真实任务、诊断和迁移验证提升能力，而不是
机械重讲。

材料存在、智能体生成的阶段总结、项目完成和测试通过都不能单独把学习进度
改成“已完成”。导入旧计划时，如果用户维护的状态与智能体总结冲突，以用户
状态为准；仍不确定时采用更保守的状态并询问用户。

证据开头注明口径：`当前验证：`、`历史观察：`、`用户陈述：`或
`材料存在：`。混合来源时逐项标明。智能体生成的代码、测试结果或课程材料
默认属于“材料存在”，不能单独提高学习者的能力等级。

## `.learning/RESOURCES.md`

```md
# 学习资料

## 一手资料

- [<标题>](<路径或 URL>)
  用于：<具体能力或问题>。

## 辅助资料

- [<标题>](<路径或 URL>)
  用于：<具体能力或问题>。

## 真实反馈

- <评审者、社区、测量方式、练习环境或评价渠道>
  用于：<反馈类型>。

## 缺口

- <目前缺少可信资料或反馈渠道的重要领域>
```

只保留少量经过注释的资料。已经证明浅薄、过时或偏离使命的内容应删除。

## `.learning/NOTES.md`

记录长期有效的教学偏好、无障碍需求、有效类比、工具约束和临时工作笔记。
教学偏好可以包括图形化强度、是否优先使用 Mermaid 或表格、源码路径和关键
代码片段的展示密度。不要在这里记录掌握程度。

## `.learning/learning-records/NNNN-<slug>.md`

```md
# <已经证明的能力或已经纠正的误解>

<学习者证明了什么、为什么重要，以及未来会话现在可以安全假设什么。>

## 证据

- <回答、任务结果、测试、观察、评审反馈或产物链接>
```

可选：

```md
## 复习

- 建议复查日期：YYYY-MM-DD
- 检索问题：<使用变化后的问题检查同一能力>
```

学习记录不是会话日志。只有证据建立了新的能力下限、纠正误解或改变
学习使命时才创建。

如果新证据推翻了旧记录，不删除旧记录，而是标记：

```md
状态：已被 learning-records/NNNN-<slug>.md 取代
```

## `.learning/reference/*`

默认使用 Markdown，保存紧凑、持久的查阅材料，例如：

- 词汇；
- 流程与检查清单；
- 图示与心智模型；
- 语法或演算示例；
- 决策启发。

参考资料记录知识，不证明学习者已经掌握。

## `.learning/lessons/*`

只有内容会被复用，或交互和视觉结构能显著改善学习时才创建课程文件。内容应
包括：

- 目标能力和可观察成果；
- 前置能力；
- 最少必要讲解；
- 一次真实尝试；
- 反馈或自检机制；
- 一个迁移任务；
- 可信资料链接；
- 下一次检索问题。

默认使用 Markdown。只有交互、模拟、可视化、布局或打印需求足以成立时，
才使用 HTML。

HTML 课程使用 `NNNN-<slug>.html` 命名，并复用 `.learning/assets/` 中的
本地 CSS/JavaScript。一个课程只围绕一个可观察成果，不把整个主题做成网站。
具体出题、反馈、证据和无障碍要求见
[交互式课程规范](interactive-lessons.md)。

## `.learning/assets/*`

只保存两个以上课程可以复用的本地组件，例如：

- `lesson.css`：统一排版、明暗主题、打印和无障碍样式；
- `quiz.js`：选择题即时反馈、重试和结果复制；
- 特定领域真正复用的模拟器或图示组件。

首次创建交互课程时，可以从技能目录的 `assets/interactive-lesson/` 复制基础
模板。不要为单个题目建立组件库，不要依赖 CDN，也不要把本地测验分数当作
掌握证据。
