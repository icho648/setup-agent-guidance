# 编写 PRD

将 PRD 写成稳定、简洁的产品契约：说明为什么做、产品应表现成什么样、什么不能破坏、如何判断完成，以及谁能改变什么。不要写成实施计划。

## 1. 遵循工作区

依次遵循：用户当前请求、适用的工作区指导文件、既有产品文档惯例、本 Skill。

起草前按需读取：

- 从工作区根目录到目标目录适用的 `AGENTS.md`。
- `CONTRIBUTING.md`、`README.md` 及其引用的需求规范。
- 产品愿景、路线图、决策记录和问题跟踪说明。
- 已有 PRD、spec、requirements、RFC 或 plan 的位置、命名、状态和结构。

只读取与当前需求和文档位置有关的资料。项目已有模板时，沿用其结构并映射本 Skill 的必需语义；缺少必需语义时增加最小子节。用户未明确要求新建且已有产品文档能承载本次契约时，只更新最小必要内容，不创建平行 PRD。若工作区禁止扩展或无法无歧义映射，停止写入并请求用户决定。

## 2. 解决产品决策

先从工作区查找事实，只询问会改变目标、范围、产品不变量、硬约束、验收基线或自治边界的决策。一次只问一个高影响问题，并给出推荐答案及影响。实现 Agent 可依据代码和工作区规范决定的内部细节不要询问。

区分：

- **事实**：已有证据支持。
- **决策**：已由有权决定的人确认。
- **假设**：尚未验证，仅供分析。
- **待定**：需要人决定，可能阻塞实施。
- **建议**：基于假设提出，不属于产品基线。

不得把假设或建议升级为正式契约，用户未提及也不代表确认。存在多种合理产品行为时，将选择保留为待定或建议；不得为了补齐 PRD 自行写成已确认决策、正式行为或 AC。决定核心契约的事项未确认时保持草稿；重要决策引用确认记录和生效基线。

## 3. 选择位置与命名

按以下顺序选择：

1. 用户指定路径。
2. 工作区明确规定。
3. 已有 PRD 的主要惯例。
4. 最接近的 spec、requirements、RFC 或 plan 惯例。
5. 均不存在时使用 `docs/prd/YYYY-MM-DD-<feature-slug>.md`。

兜底日期使用执行环境本地日期。已有工作区约定时不强制日期或 `docs/prd/`。

候选位置仅有命名差异时选择既有惯例；若代表不同生命周期或受众，先询问用户。

文件存在且用户要求修改时更新；用户要求新建时不得覆盖，应询问修订现有文件还是另定名称。不要自动添加 `-2`、`final` 或 `new`。

## 4. 建立产品契约

阅读 [PRD 参考模板](references/prd-template.md)。最终文档必须包含以下语义，章节名可适配项目：

1. **问题、用户与依据**：为什么做，哪些信息已确认。
2. **目标结果与成功标准**：希望产生什么变化，不列功能。
3. **范围与预期产品行为**：关键情境下的可观察行为及非目标。
4. **产品不变量与硬约束**：演进时不得破坏的产品、业务、安全和兼容边界。
5. **验收规则**：独立、可观察、可测试或可由人工明确判断的完成标准。
6. **演进与自治边界**：实施 Agent 可自主决定什么，哪些变化必须由人确认。

开头另写一分钟摘要，但只压缩正文，不引入新需求、约束或决策。

## 5. 保证可读、可追溯、可演进

- 每类信息只在一个正文位置完整说明：问题不带方案，目标不列功能，产品行为不写内部实现，验收不重复背景。
- 需要交叉引用时使用中文名称加普通序号，如“目标 1”“产品行为 1”“产品不变量 1”。只保留行业通用的 `AC` 表示验收规则，不创造字母前缀。
- 正式条目必须源于用户请求、工作区事实或已确认决策；无来源内容放入“建议”，不得进入正式 AC。
- 每条产品行为对应目标；每条产品行为、产品不变量和硬约束至少被一个正式 AC 覆盖；AC 使用中文名称和序号反向引用对应条目。
- 通用工程规则只引用工作区文件；文件结构、类设计、任务顺序和常规技术选择留给实施 Agent。
- 删除不适用的可选内容，不编造指标、日期、调研结论或外部承诺。
- 修订时保留仍有效编号并概括实质变化。改变目标、范围、不变量、约束或验收基线时，记录产品决策并形成新基线。

存在高影响待定、核心建议或验收缺口时标记为“草稿”；契约已确认且足以独立指导实现和验收时标记为“可交付实施”。工作区规定其他状态时服从工作区。

## 6. 写入与交付

创建、完善、转换或修订时，除非用户明确只要聊天内容，否则默认写入 Markdown 文件。用户只要求评审时保持只读，除非明确要求写回。

写入前检查六项语义、信息来源、正式 AC 覆盖和待定项阻塞性。交付时返回：文档路径、一句话目标、路径与模板依据、高影响待定项和文档状态。
